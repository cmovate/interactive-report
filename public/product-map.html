Product Overview

What El-Via ABM is, who it's for, and how it works

Core premise: El-Via ABM lets SDRs and CMOs run LinkedIn outreach campaigns at scale — automatically sending connection requests, sequenced messages, profile views, company follows, and post likes — while surfacing intelligence about warm leads at target accounts.

🔗¬ü¬é¬Ø Primary Use Case

Outbound LinkedIn ABM: define a list of target companies + personas, build a message sequence, and let El-Via automate the full outreach funnel — from initial connection request through multi-step follow-up — while respecting LinkedIn daily limits.

🔗¬ü¬ë¬§ Who Uses It

SDRs, AEs, and fractional CMOs managing LinkedIn outreach for one or more companies. Each company gets its own Workspace with its own LinkedIn accounts, campaigns, and contacts. Multiple accounts can run per workspace.

🔗¬ü¬î¬å LinkedIn Connection

All actions go through Unipile API — a middleware wrapping LinkedIn's internal APIs. Each account is authenticated once in Unipile and accessed via account_id. El-Via never stores LinkedIn passwords.

√¢¬ö¬ô√Ø¬∏¬è How Automation Works

Background workers run on setInterval loops (every 30 – 60 min). Each cycle processes a small batch of contacts per account, respecting per-account daily limits. All actions are rate-limited and logged in the database.

Campaigns

Each campaign targets a contact list and runs a full LinkedIn outreach sequence: connection request √¢¬Ü¬í approval √¢¬Ü¬í follow-up messages. Multiple campaigns run simultaneously across different accounts.

Campaign Properties
  • Name — internal label

Inbox

Mirrors the LinkedIn inbox for connected accounts. Shows all conversations, flags replies from campaign contacts, and uses AI to classify replies as positive / neutral / negative with next-step recommendations.

Features
  • Full conversation thread view
  • Contact matching to campaign records
  • Reply classification: positive / neutral / negative

  • conversationAnalyzer — AI classification of intent
  • messageSender — sends next sequence step after reply

Background Automations

16 workers running continuously ‚Äî full engine of El-Via

All automations start via module.start() in server.js after DB init. Run on Railway ‚Äî no external cron needed. Manual triggers available for all via API endpoints.
🚀 Outbound
Every 15min
invitationSender.js
Sends LinkedIn connection requests per active campaign. Respects working hours + daily limit per account (settings.limits.connection_requests). Uses provider_id (ACoXXX), not vanity slug. Manual: POST /api/campaigns/:id/scrape-engagement
Every 5min
messageSender.js
Sends LinkedIn messages by campaign sequence. Supports A/B/C variants, delay scheduling (days/hours/minutes), working hours. Jitter ¬±hours on scheduling to avoid patterns. Max 3 concurrent sends.
Every 15min
profileViewer.js
Views LinkedIn profiles of campaign contacts. Max once every 7 days per contact. 30‚Äì90s random delay between views. Daily limit from account settings. Warm signal before messaging.
Every 30min
companyFollowSender.js
Sends company page follow invites to connected contacts. State machine: normal ‚Üí waiting_5d ‚Üí drip ‚Üí waiting_7d. Monthly limit: 250/account. Batch: 5. Requires company_page_urn in account settings. Manual: POST /api/company-follow/run
Daily (timed)
withdrawSender.js
Withdraws pending connection requests not accepted within withdraw_after_days. Runs once daily at configured time. Frees up pending invite slots.
Every 1h
likeSender.js
Likes posts/comments of engaged contacts from personal accounts. Skips contacts already liked by company page today. Dedup via liked_ids. Only contacts with engagement_level: engaged/average_engaged.
üì• Inbound & Tracking
Every 30min
approvalChecker.js
Fetches full relations list from Unipile for each active account. Marks invite_approved=true when contact's provider_id appears. Also marks already_connected for pre-existing connections not in any campaign.
Every 30min
inboxPoller.js
Backup webhook insurance. Fetches last 15 chats per account, syncs to inbox_threads/inbox_messages. Marks msg_replied on campaign contacts who replied ‚Äî catches missed webhooks. Manual: POST /api/inbox/poll
08:00 / 14:00 / 20:00
profileViewScraper.js ‚Äî Views
Fetches last 5 LinkedIn profile viewers per account via Voyager API. Stores in profile_view_events. Identified viewers matched to contacts/lists. Adds unknown viewers to "Profile Viewers" list. Then auto-enriches company names via Unipile work_experience API. Manual: POST /api/profile-views/scrape
08:00 / 14:00 / 20:00
profileViewScraper.js ‚Äî Posts
Fetches latest 10 posts per account via Unipile GET /api/v1/users/{pid}/posts. Upserts to user_posts table ‚Äî updates likes/comments/reposts/impressions on re-run. All workspaces. Manual: POST /api/posts/scrape
Every 30min
profileViewScraper.js ‚Äî Engagement Sync
Incremental sync of reactions + comments per user post. Fetches 10 at a time, stops on first known item. If all-new ‚Üí fetches 10 more with cursor. Saves to post_reactions + post_comments tables. Manual: POST /api/posts/engagement/sync
Every 30min
followerScraper.js
Scrapes company page followers. Learning mode (days 0‚Äì7): 3×/day full scrape. Incremental after: top N pages, stops when all known. Cross-references with contacts to mark company_follow_confirmed. Requires company_page_urn in account settings. Manual: POST /api/followers/scrape
üßÝ Intelligence & Enrichment
Daily (2×)
engagementScraper.js
3 phases: (1) Scrape posts+comments for contacts, save JSONB. (2) Personal-account likes if enabled ‚Äî skips company-liked today. (3) Classify engagement_level per contact: un_engaged / average_engaged / engaged. Feeds Feed tab + likeSender.
Every 1h
companyEngagementScraper.js
Scrapes company posts (last 14 days) for every company in campaign_companies + opportunity_companies. Up to 50 posts per company. Whether to LIKE is controlled separately by campaign settings.
Every 1h
opportunityScraper.js
Finds 1st-degree connections at target companies via Unipile search (DISTANCE_1). Sources: campaign_companies + opportunity_companies + contacts' work_experience. Saves to contacts with already_connected=true, campaign_id=NULL. Powers Opportunities tab.
Every 6h
scheduler.js
Auto-scan: scans all list companies for 1st-degree connections every 6 hours. Populates Opportunities page with new people to engage.
Event-driven
conversationQueue.js + conversationAnalyzer.js
2-minute debounce per contact after message received. Max 3 concurrent Claude AI analyses. Detects positive/negative intent, suggests reply. Restart recovery re-queues unanalyzed contacts. Triggered by webhook or inboxPoller.
Daily
statsSnapshotter.js
Saves daily funnel snapshots per campaign to campaign_daily_stats. Idempotent (ON CONFLICT DO UPDATE). Runs on startup to catch missed days, then every 24h.

End-to-End Outreach Flow

From contact added to deal booked

Day 0 — Contact Added
  • Imported via CSV / search / opportunityScraper
  • Enqueued for enrichment √¢¬Ü¬í profile_data + provider_id set
  • If already_connected=true √¢¬Ü¬í skip invite, go to messages
Day 1 – 3 — Invite Phase
  • invitationSender sends connection request (with optional note)
  • profileViewer views their profile (warm signal)
  • invite_sent=true recorded
Day 3 – 7 — Approval
  • Webhook or polling detects connection accepted
  • invite_approved=true set
  • companyFollowSender sends company page follow
Day 7+ — Message Sequence
  • messageSender sends Message 1 (first follow-up)
  • Delay N days √¢¬Ü¬í Message 2, 3... up to 20 steps
  • conversationQueue monitors for replies √¢¬Ü¬í msg_replied=true
  • conversationAnalyzer classifies reply sentiment + intent
  • positive_reply=true √¢¬Ü¬í surfaces in Inbox for human follow-up
Parallel — Engagement
  • engagementScraper classifies contact activity level
  • likeSender warms engaged contacts with post likes
  • Feed tab shows what contacts are posting about
Always Running — Intelligence
  • opportunityScraper adds 1st-degree connections at target companies
  • followerScraper surfaces organic followers as warm leads
  • statsSnapshotter tracks growth over time
Key Insight: El-Via is not a blast-and-hope tool. The intelligence layer (Opportunities + Feed + Inbox AI) surfaces the right moment to engage — when a target is active on LinkedIn, just changed jobs, or is already a 1st-degree connection at a company you're targeting.
