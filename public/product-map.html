Product Overview

What El-Via ABM is, who it's for, and how it works

Core premise: El-Via ABM lets SDRs and CMOs run LinkedIn outreach campaigns at scale √¢¬Ä¬î automatically sending connection requests, sequenced messages, profile views, company follows, and post likes √¢¬Ä¬î while surfacing intelligence about warm leads at target accounts.

√∞¬ü¬é¬Ø Primary Use Case

Outbound LinkedIn ABM: define a list of target companies + personas, build a message sequence, and let El-Via automate the full outreach funnel √¢¬Ä¬î from initial connection request through multi-step follow-up √¢¬Ä¬î while respecting LinkedIn daily limits.

√∞¬ü¬ë¬§ Who Uses It

SDRs, AEs, and fractional CMOs managing LinkedIn outreach for one or more companies. Each company gets its own Workspace with its own LinkedIn accounts, campaigns, and contacts. Multiple accounts can run per workspace.

√∞¬ü¬î¬å LinkedIn Connection

All actions go through Unipile API √¢¬Ä¬î a middleware wrapping LinkedIn's internal APIs. Each account is authenticated once in Unipile and accessed via account_id. El-Via never stores LinkedIn passwords.

√¢¬ö¬ô√Ø¬∏¬è How Automation Works

Background workers run on setInterval loops (every 30√¢¬Ä¬ì60 min). Each cycle processes a small batch of contacts per account, respecting per-account daily limits. All actions are rate-limited and logged in the database.

Each campaign targets a contact list and runs a full LinkedIn outreach sequence: connection request √¢¬Ü¬í approval √¢¬Ü¬í follow-up messages. Multiple campaigns run simultaneously across different accounts.

Background Automations

All workers running continuously √¢¬Ä¬î the engine of El-Via

All automations registered in server.js via module.start() after DB schema init. Run on Railway server process √¢¬Ä¬î no external cron needed.
Outreach Automations
Every 45min
invitationSender.js
Sends LinkedIn connection requests to uninvited contacts. Respects daily limit. Attaches note if configured. Sets invite_sent=true.
Every 30min
messageSender.js
Sends next message in sequence to approved contacts. Checks delay conditions between steps. Updates msg_N_text, msg_N_sent_at, msg_sent=true.
On-demand
withdrawSender.js
Withdraws unanswered connection requests older than 3 weeks. Frees up pending invite slots. Triggered manually from campaign UI.
Every 60min
companyFollowSender.js
Sends company page follows to connected contacts. Uses company_page_url from account settings. Monthly cap: 250. Sets company_follow_invited=true.
Every 60min
profileViewer.js
Views profiles of contacts as a warm signal before messaging. Sets last_profile_view_at. Respects daily view limit.
Engagement Automations
On-demand
engagementScraper.js
Fetches recent posts for all campaign contacts. Classifies: un_engaged / average_engaged / engaged. Stores engagement_data JSON. Feeds likeSender and Feed tab.
On-demand
companyEngagementScraper.js
Fetches posts from company LinkedIn pages (not individuals). Data flows to Feed tab.
On-demand
likeSender.js
Likes recent posts/comments of engaged contacts. Creates warm touchpoints. Tracks post_likes_sent and comment_likes_sent per contact.
Intelligence Automations
Every 60min
opportunityScraper.js
Scans all 3 company sources (campaign_companies, opportunity_companies, contacts.work_experience) for 1st-degree connections (DISTANCE_1). Saves to contacts with already_connected=true, campaign_id=NULL. Powers Opportunities tab.
Every 60min
followerScraper.js
Fetches new followers of connected accounts. Identifies organic warm leads.
Every 6 hours
statsSnapshotter.js
Snapshots key LinkedIn metrics per account (connections count, followers). Builds trend data for Admin Analytics.
Inbox Automations
Every 30min
conversationQueue.js
Polls Unipile for new messages on all accounts. Matches threads to contacts. Updates msg_replied=true and conversation_id.
Every 30min
conversationAnalyzer.js
Sends unanalyzed conversation threads to AI. Classifies intent (positive/neutral/negative), extracts what person is asking, generates recommended next step.

End-to-End Outreach Flow

From contact added to deal booked

Day 0 √¢¬Ä¬î Contact Added
  • Imported via CSV / search / opportunityScraper
  • Enqueued for enrichment √¢¬Ü¬í profile_data + provider_id set
  • If already_connected=true √¢¬Ü¬í skip invite, go to messages
Day 1√¢¬Ä¬ì3 √¢¬Ä¬î Invite Phase
  • invitationSender sends connection request (with optional note)
  • profileViewer views their profile (warm signal)
  • invite_sent=true recorded
Day 3√¢¬Ä¬ì7 √¢¬Ä¬î Approval
  • Webhook or polling detects connection accepted
  • invite_approved=true set
  • companyFollowSender sends company page follow
Day 7+ √¢¬Ä¬î Message Sequence
  • messageSender sends Message 1 (first follow-up)
  • Delay N days √¢¬Ü¬í Message 2, 3... up to 20 steps
  • conversationQueue monitors for replies √¢¬Ü¬í msg_replied=true
  • conversationAnalyzer classifies reply sentiment + intent
  • positive_reply=true √¢¬Ü¬í surfaces in Inbox for human follow-up
Parallel √¢¬Ä¬î Engagement
  • engagementScraper classifies contact activity level
  • likeSender warms engaged contacts with post likes
  • Feed tab shows what contacts are posting about
Always Running √¢¬Ä¬î Intelligence
  • opportunityScraper adds 1st-degree connections at target companies
  • followerScraper surfaces organic followers as warm leads
  • statsSnapshotter tracks growth over time
Key Insight: El-Via is not a blast-and-hope tool. The intelligence layer (Opportunities + Feed + Inbox AI) surfaces the right moment to engage √¢¬Ä¬î when a target is active on LinkedIn, just changed jobs, or is already a 1st-degree connection at a company you're targeting.
